# Contracts

Minimal bulletin-board contract for the cross-group binding research artifact.

## Core components

- `CrossGroupBallotBox.sol` — bulletin-board contract with bound-ballot acceptance,
  committee-authenticated share submission, and canonical fixed-width ciphertext enforcement.
- `Groth16Verifier.sol` — **real** Groth16 verifier generated from the compiled circuit
  (16,494 constraints, 36 public inputs). This is not a mock.
- `MockVerifier.sol` — test-only stub used by unit tests that focus on board logic.
- `MockBallotValidityVerifier.sol` — test-only ballot-validity stub.
- `NaiveBallotBox.sol` — deliberately **insecure** contract without limb checks,
  used only for the attack demonstration.

## Key invariants

- Ciphertext must be exactly 768 bytes (canonical fixed-width serialization).
  This is required for injectivity of the limb decomposition.
- 32 limbs are extracted from the posted ciphertext bytes and compared against
  the public inputs before proof verification.
- Committee members must be registered before election start; only members can submit shares.
- Threshold must be ≤ committee size at election init time.

## Test suites

- `CrossGroupBallotBox.test.ts` — 15 unit tests (mock verifier, board-level invariants)
- `EndToEnd.test.ts` — 5 tests with real Groth16 proofs generated by snarkjs
- `AttackDemo.test.ts` — 3 tests demonstrating naive-accepts / bound-rejects with real verifier
- `PaillierE2E.test.ts` — 1 test: real @cgbl/crypto threshold Paillier (512-bit, 2-of-3) → 2× encrypt → 2× canonical serialize → 2× real Groth16 proof → 2× on-chain castBallot → closeVoting → publishAggregate → 2× submitShare → finalize → threshold decrypt

## Commands

```bash
yarn quick:attack
yarn workspace @cgbl/contracts compile
yarn workspace @cgbl/contracts test
cd ../../ && yarn gas:bench
REAL_VERIFIER=true yarn workspace @cgbl/contracts deploy:local
```

`yarn gas:bench` is the authoritative path for production-width gas numbers.
The gas values printed from `EndToEnd.test.ts` are a smoke trace for core
operations and include a deliberately tiny mock share payload.
